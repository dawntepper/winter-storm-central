# Location seed data

## Files

| File | Purpose |
|------|---------|
| `uscities.csv` | SimpleMaps US cities (population, county FIPS, ZIPs). [Source](https://simplemaps.com/data/us-cities) |
| `us-places-top50.json` | Top 50 cities per state — generated by `npm run db:build:places` |
| `city-priority-additions.json` | CDPs / unincorporated places missing population in SimpleMaps |
| `city-county-fips-cache.json` | FCC geocoding cache for cities without county FIPS |
| `zip-cache.json` | Primary ZIP per catalog city (zippopotam.us + manual fallbacks) |

## City catalog sources (see `scripts/db/lib/load-sources.js`)

Cities are merged in precedence order:

1. `scripts/db/data/us-places-top50.json` — top 50 by population per state
2. `STATES_AND_CITIES` in `src/components/ZipCodeSearch.jsx` — 3–4 major cities per state
3. `FORECAST_PICKER_FILL` in `src/data/cityCatalog.js` — one major city per uncovered state
4. `src/content/cities/*.json` — rich alert pages (`has_static_page=true`, county FIPS)

## Regenerate population dataset

```bash
# Download latest SimpleMaps CSV (optional refresh)
curl -L -o scripts/db/data/uscities.csv https://simplemaps.com/static/data/us-cities/uscities.csv

npm run db:build:places
```

## Seed commands

```bash
npm run db:build:county-cache   # resolve ~16 remaining county gaps via FCC
npm run db:seed:locations       # full pipeline + per-state counts
npm run db:seed:zips -- --offline   # use cached ZIPs only (no network)
```

After changing any source, re-seed: `npm run db:seed:locations`.
